# Бриф на разработку: Tennis School Mini App

> Версия: 1.0 (MVP)
> Дата: 2 марта 2026
> Основа: research.md — исследование рынка и экспертная оценка

---

## Концепция

Telegram Mini App — каталог услуг школы тенниса.
Клиент открывает бота → нажимает кнопку → видит услуги с ценами → записывается в 3 тапа.

**Ключевой принцип:** выглядит и ощущается как нативное мобильное приложение, а не как сайт или чатбот.

---

## Технический стек

| Слой | Технология |
|---|---|
| Фронтенд | React 19 + Vite + TailwindCSS 4 |
| Telegram SDK | `@telegram-apps/sdk-react` |
| Навигация | React Router v7 (stack) |
| Бэкенд | Python FastAPI + SQLAlchemy |
| БД | PostgreSQL |
| Бот | python-telegram-bot (webhook) |
| Деплой | Docker Compose (nginx + api + db) |
| Изображения | WebP, lazy-load, CDN |

**Целевой размер билда:** < 200KB gzipped

| # | Элемент | Тип | Содержимое |

| 2 | Чипы категорий | Горизонтальная строка кнопок | "Все", "Индивидуальные", "Групповые" |
| 3 | Карточка услуги (×N) | Тапабельная карточка | Фото (80×80, WebP, rounded 12px), название, длительность, цена, индикатор мест |
| 4 | Индикатор мест | Текст hint-color | "Осталось N" (показывать если мест ≤5) |
| 5 | Скелетон | Анимированные плейсхолдеры | 3 карточки-скелетона при загрузке |

| Действие | Что происходит |

| Тап на карточку услуги | Переход на экран 2 (детали) |
| Скролл вниз | Прокрутка списка (все карточки загружены сразу — каталог небольшой) |

### Telegram API

| API | Использование |
|---|---|
| `BackButton` | Скрыт (это корневой экран) |
| `MainButton` | Скрыт |
| `WebApp.ready()` | Вызвать после рендера первой карточки |
| `WebApp.expand()` | Вызвать при монтировании — раскрыть на весь экран |
| `themeParams` | Цвета карточек, фона, текста из темы |

### Элементы

| # | Элемент | Тип | Содержимое |

### Действия

| Действие | Что происходит |
|---|---|
| Тап "Отправить номер" | `requestContact()` → автозаполнение поля телефона. HapticFeedback: `impactOccurred('light')` |
| Ввод телефона вручную | Маска ввода, валидация |
| Ввод комментария | Свободный текст |
| Тап BackButton | Возврат на экран 3 |
| Тап MainButton "Записаться" | Отправка заявки на бэкенд → переход на экран 5. MainButton показывает лоадер во время запроса |

### Telegram API

| API | Использование |
|---|---|
| `BackButton` | Видим |
| `MainButton` | Текст: "Записаться". При отправке: `showProgress()` + disable |
| `requestContact()` | По тапу на кнопку получения телефона |
| `HapticFeedback` | `notificationOccurred('success')` при успешной отправке |
| `initDataUnsafe.user` | `first_name`, `last_name` для авто-заполнения |

### Валидация

| Поле | Правило | Ошибка |
|---|---|---|
| Телефон | Обязательно. Формат: +7XXXXXXXXXX или через requestContact | "Укажите номер телефона" |
| Комментарий | Необязательно. Макс 200 символов | — |

  --bg:            var(--tg-theme-bg-color);

  --text:          var(--tg-theme-text-color);

  --section-bg:    var(--tg-theme-section-bg-color);

### Состояния загрузки

Каждый экран, загружающий данные, показывает скелетон:
- **Каталог:** 3 карточки-скелетона (серые прямоугольники с pulse-анимацией)
- **Детали:** блок-фото (серый прямоугольник) + 3 строки текста
- **Дата/время:** календарь-скелетон + 5 чипов-скелетонов
- **Подтверждение:** нет скелетона (данные уже в памяти)

### Пустые состояния

| Экран | Когда | Что показать |
|---|---|---|
| Каталог | Нет услуг (невозможно, но на всякий случай) | Иконка теннисной ракетки + "Услуги скоро появятся" |
| Дата/время | Нет слотов на выбранную дату | "На эту дату нет свободных слотов" + "Выберите другой день" |
| Дата/время | Нет слотов вообще | "Сейчас нет свободных мест. Попробуйте позже" |

### Ошибки

| Ситуация | UI |
|---|---|
| Сеть недоступна | Полноэкранное: иконка + "Нет соединения" + кнопка "Повторить" |
| Ошибка сервера | `WebApp.showAlert("Что-то пошло не так. Попробуйте ещё раз")` |
| Слот занят между выбором и подтверждением | `WebApp.showPopup({ message: "Это время уже занято. Выберите другое", buttons: [{ type: "ok" }] })` → возврат на экран 3 |

## Модель данных (MVP)

```
services
├── id: uuid
├── title: string           "Индивидуальная тренировка"
├── description: text        "Персональное занятие с тренером..."
├── duration_minutes: int    60
├── price: int               3000  (в рублях, 0 = бесплатно)
├── category: enum           "individual" | "group" | "trial"
├── image_url: string        "/images/individual.webp"
├── max_participants: int    1 (для индивид.) или 6 (для групповых)
├── is_active: boolean
└── sort_order: int

coaches
├── id: uuid
├── name: string             "Иван Петров"
├── title: string            "КМС по теннису"
├── experience_years: int    10
├── photo_url: string
└── is_active: boolean

schedule_slots
├── id: uuid
├── service_id: FK → services
├── coach_id: FK → coaches
├── date: date               2026-03-15
├── start_time: time         10:00
├── end_time: time           11:00
├── max_participants: int    (наследуется от service или override)
├── booked_count: int        0
└── is_available: boolean    (computed: booked_count < max_participants)

bookings
├── id: uuid
├── booking_number: int      1042 (автоинкремент, для отображения)
├── slot_id: FK → schedule_slots
├── telegram_user_id: bigint
├── user_first_name: string
├── user_last_name: string
├── user_phone: string
├── comment: text
├── status: enum             "confirmed" | "cancelled"
├── created_at: timestamp
└── updated_at: timestamp
```

## API-эндпоинты (MVP)

```
GET  /api/services                    → список услуг с фильтром по категории
GET  /api/services/:id                → детали услуги + тренер + ближайший слот
GET  /api/services/:id/slots?date=    → доступные слоты на дату
GET  /api/slots/dates?service_id=     → даты с доступными слотами (для календаря)
POST /api/bookings                    → создание записи
```

Каждый запрос валидирует `initData` (HMAC-SHA256 с bot token).
