> Файлы: `tg-app/index.html`, `tg-app/styles.css`, `tg-app/app.js`

## 1. Запуск проекта

### 1.1. Локально (в браузере, без Telegram)

```bash
cd tg-app
npx serve .
# или
python -m http.server 8080
```

Открыть `http://localhost:8080` (или `http://localhost:3000` для `serve`).

> **Важно:** вне Telegram объект `window.Telegram.WebApp` недоступен. Приложение работает с ограничениями:
> - BackButton и MainButton Telegram не отображаются
> - HapticFeedback не работает
> - Имя пользователя отображается как «Гость»
> - `tg.close()` не вызывается
> - Алерты показываются через нативный `alert()` вместо `tg.showAlert()`

### 1.2. В Telegram (через ngrok)

```bash
# Терминал 1: запуск сервера
cd tg-app && npx serve . -l 8080

# Терминал 2: туннель
ngrok http 8080
```

1. Скопировать HTTPS-ссылку из ngrok (например `https://xxxx.ngrok-free.app`)
2. Открыть @BotFather в Telegram
3. `/mybots` → выбрать `@Pervaya_school_tennis_Bot` → Bot Settings → Menu Button → ввести URL
4. Открыть бота → нажать кнопку меню → приложение запустится внутри Telegram

### 1.3. Production (Vercel)

URL приложения: `https://tennis-school-telegram.vercel.app/`
Бот: `@Pervaya_school_tennis_Bot`
Deeplink: `https://t.me/Pervaya_school_tennis_Bot?start=from_app`

## 2. Структура экранов

Приложение содержит **8 экранов** (div-ы внутри `#app`):

| # | ID экрана | Назначение | Таб-бар |
|---|-----------|------------|---------|
| 0 | `screen-onboarding` | Приветствие (показывается 1 раз) | Скрыт |
| 1 | `screen-catalog` | Каталог услуг (главный) | Виден |
| — | `screen-bookings` | Мои записи | Виден |
| — | `screen-contacts` | Контакты | Виден |
| 2 | `screen-detail` | Детали услуги | Скрыт |
| 3 | `screen-datetime` | Выбор даты и времени | Скрыт |
| 4 | `screen-confirm` | Подтверждение записи | Скрыт |
| 5 | `screen-success` | Успешная запись | Скрыт |

Дополнительно: **модалка-оффер** (`#offer-overlay`) — предложение подписки на бота (показывается 1 раз).

### Навигация

```
Онбординг → Каталог → Детали → Дата/Время → Подтверждение → Успех
                ↕           ↑          ↑              ↑
           Записи       (назад)    (назад)        (назад)
           Контакты
```

- Вперёд: slide-анимация вправо (300ms)
- Назад: slide-анимация влево (300ms)
- Табы: переключение между Каталог / Записи / Контакты

### 3.4. Расписание

- Генерируется автоматически при загрузке (`generateSlots()`)
- Период: 14 дней вперёд от текущей даты
- Возможные времена: `09:00, 10:00, 11:00, 12:00, 14:00, 15:00, 16:00, 17:00, 18:00, 19:00`
- На каждый день для каждой услуги: 3–6 случайных слотов
- 20% дней случайно пропускаются (нет слотов)
- Для текущего дня: прошедшие слоты отфильтровываются

## 5. Известные баги и проблемы

### BUG-01: Расписание нестабильно — каждая перезагрузка генерирует новые слоты

**Файл:** `app.js:107-148`
**Проблема:** `generateSlots()` использует `Math.random()`, поэтому при каждой перезагрузке страницы доступные даты и времена меняются. Это может сбивать пользователя.

**Примечание:** Это по дизайну для MVP (данные захардкожены), но следует учитывать при тестировании — конкретные слоты не воспроизводимы.

### BUG-02: Подтверждение закрытия на экране успеха

**Файл:** `app.js:18`
**Проблема:** `tg.enableClosingConfirmation()` вызывается всегда, включая экран успеха. На экране успеха пользователь уже завершил действие, и подтверждение при закрытии не нужно.

### BUG-03: `sendData` закомментирован

**Файл:** `app.js`
**Проблема:** `tg.sendData(JSON.stringify(data))` закомментирован. Данные записи не отправляются боту, только логируются в консоль и сохраняются в localStorage.

**Примечание:** Это ожидаемое поведение для MVP (нет бэкенда), но подсказка «Мы отправили подтверждение в чат бота» вводит в заблуждение.

## 6. Проверки в Telegram

### 6.1. Telegram API (только в Telegram)

| Проверка | Как |
|----------|-----|
| `tg.expand()` | Mini App раскрывается на весь экран |
| `tg.ready()` | Вызывается после рендера каталога (~400ms) |
| BackButton | Появляется/скрывается на нужных экранах (см. TC-15) |
| MainButton | Текст и состояние меняются (см. TC-16) |
| MainButton progress | На экране подтверждения при отправке: спиннер на кнопке ~300ms |
| HapticFeedback | Вибрация при: навигации (impact/light), выборе чипов (selection), успехе (notification/success) |
| themeParams | Цвет MainButton = `button_color` из темы |
| Closing confirmation | При попытке закрыть Mini App — подтверждение |

### 6.2. Тёмная тема

Все цвета задаются через CSS-переменные Telegram (`--tg-theme-*`). При переключении темы в Telegram:

| Элемент | Проверить |
|---------|-----------|
| Фон страницы | `--tg-theme-bg-color` |
| Текст | `--tg-theme-text-color` |
| Подсказки | `--tg-theme-hint-color` |
| Карточки | `--tg-theme-section-bg-color` |
| Accent (чипы, выбор) | `--tg-theme-accent-text-color` |
| Разделители | `--tg-theme-section-separator-color` |
| Кнопки | `--tg-theme-button-color` / `--tg-theme-button-text-color` |
| Ошибки | `--tg-theme-destructive-text-color` |

### 6.3. Safe Area

CSS использует переменные:
- `--tg-safe-area-inset-top` — отступ для вырезов (notch)
- `--tg-content-safe-area-inset-top` — отступ контента
- `--tg-safe-area-inset-bottom` — отступ снизу (home indicator)

Проверить на устройствах с вырезами (iPhone с notch, Dynamic Island).

## 7. Граничные случаи

| Случай | Как проверить | Ожидание |
|--------|---------------|----------|
| Нет доступных слотов на дату | Выбрать день без синей точки (или подождать конец 14-дневного окна) | Сообщение: «Нет свободных слотов на эту дату» |
| Все слоты на сегодня прошли | Открыть приложение вечером | Сегодняшние прошедшие слоты не показываются |
| Смена месяца в календаре | Стрелка ► для следующего месяца | Корректное переключение, но не дальше +1 месяца |
| Стрелка ◄ на текущем месяце | Нажать ◄ | Заблокирована (disabled) |
| Длинное имя из Telegram | Имя > 30 символов | Не обрезается, но не должно ломать layout |
| Комментарий = 200 символов | Ввести ровно 200 символов | Принимается, больше 200 — `maxlength` не даёт ввести |
| Двойной тап на MainButton | Быстро нажать «Записаться» дважды | Не должно создаваться 2 записи (кнопка disable после нажатия) |
| localStorage заполнен | Много записей | Приложение не падает |
| Нет `window.Telegram` | Открыть в обычном браузере | Приложение работает с fallback-поведением |
